git and GitHub

If you don’t already have a GitHub account, head over to https://github.com/join and create an account. You can use this account throughout the course to work on assignments.

You’ll also need to make sure you have git installed on your computer. Go to GitHub’s instructions for setting up and configuring git on your machine, and follow the steps there to install git (if you didn’t already have it installed) and configure git with your GitHub username and email address.

Getting Online

Okay, it’s time to push your project0 folder to a repository of your own on GitHub! In your terminal window, navigate to your project0 directory, and then run:

$ git init
$ git add -A

to let git know that you want to include all changed files (-A) in your next commit to this repository. Now, run:

$ git commit -m "My first webpage"

to commit your changes to this repository. The string after -m is your commit message, a short written description of the changes you’ve made in this commit. Writing succinct, informative commit messages will help you refer back to old changes later!

Now, let’s push our changes to the internet. Navigate to GitHub, logging into your account if necessary, and then at the top right next to your avatar there should be a dropdown menu with a + symbol. Click on that symbol and from the dropdown select New repository. On the page that appears, type in project0 as your repository name, provide an optional description if you wish, select Public as your visibility option, and leave the Initialize this repository with a README checkbox unchecked; recall that we already have a file called README.md in our staging area! When all finished, click Create repository.

Almost there! Now jump back your terminal, and run the following:

$ git remote add origin https://github.com/USERNAME/project0.git
$ git push -u origin master

Only use the full git push command above the very first time you push to the remote repository, since it is a shorthand to set the upstream branch you want to push to (recall what this means from Lecture 1). Afterwards, when you update, you can use the more-familiar:

$ git push

You may be prompted to enter your GitHub username and password before doing so. (If you’ve used GitHub before and your profile is configured to use SSH keys, see here for more information. But once finished, your commit should be pushed to GitHub. Substituting your own username for USERNAME, confirm as much by visiting http://github.com/USERNAME/project0.

Now there’s just one more step within GitHub to enable your code deployed to GitHub Pages. Near the top of your repository page on GitHub you should find a link to Settings. Click it, and then scroll down to the GitHub Pages section. Under Source in that section, select master branch from the dropdown menu, then click Save. Now open a new browser tab and visit http://USERNAME.github.io/project0. You should see a webpage that just says “Hello, world!” with a title of “My Webpage.” Your webpage is now deployed to the internet… congratulations!
